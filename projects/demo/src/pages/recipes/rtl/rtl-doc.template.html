These examples show how masked inputs behave in right-to-left contexts.

Date and phone masks often mix digits, separators, placeholders, and text. In native dir="rtl" inputs, browsers may visually reorder such values according to bidirectional text rules.

The actual input value can still be correct, even if the displayed text looks unexpected.

The examples below show native RTL behavior and possible workarounds.

Unicode direction marks are used here as an experimental workaround. They can improve the visual order, but may temporarily appear in the raw input value while editing.

A date placeholder like 12/mm/yyyy contains numbers, slashes, and Latin letters. In dir="rtl" inputs, browsers may display this mixed-direction text in an unexpected order.
